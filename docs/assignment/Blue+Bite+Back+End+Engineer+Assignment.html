Blue Bite Back End Engineer Assignment

This is the engineering homework assignment for the Senior Back End Engineering position. Your solutions to the problems in this assignment will help us understand your engineering and database design skills. Please follow the instructions carefully.

Introduction

Imagine a system which needs to serve payloads for millions of different smart tags. Each tag can hold multiple metadata parameters describing the object it's attached to. The set of parameters is not consistent across all tags because each tag can be attached to a different item (e.g. a shoe, a soccer ball, or a bag). Imagine we have multiple external vendors who are sending us JSON payloads containing updated metadata for the tags as well as new tags' information. Each payload can contain metadata for anywhere from 0 to 10,000 tags.

Data Constraints

There is a JSON Schema that describes the format of incoming payloads. It can be found here. You may use it to validate incoming payloads.

Each tag metadata record will be a part of an array and will look something like this:

{
    "tag_id": "0fadf66d045648bfa880d7d07af203bb",
    "metadata": [
        {
            "key": "type",
            "value": "shoe"
        },
        {
            "key": "color",
            "value": "blue"
        },
        {
            "key": "handedness",
            "value": "right"
        }
    ]
}

Metadata keys and values are always strings. Example payloads:

  1. 5d207da03b0040578e4c5160597357b7
  2. 7447156584c543658455558747c64d2c
  3. 995c3f51ebd7486695d8947152bb38d3
  4. a3e3853750724e2994515bb70d646c32

Application Functionality Constraints

  1. The application should accept JSON payloads via HTTP RESTful routes.
    • when the application receives a new payload it must be validated, parsed and (if the data is correct) stored in the database
    • the application must be able to handle validation errors. The format of the errors is your choice so long as it is consistent and adheres to HTTP protocol standards.
  2. The application must offer a way to query the database in order to retrieve tags from the database and serve them as a JSON payload filterable by query. It should:
    • validate the incoming query.
    • return a list of tags matching the requested criteria.
  3. Authentication capabilities are not required.

Example Query

The following query should return all tags with key color and value blue in their metadata:

{
    "key": "color",
    "value": "blue"
}
